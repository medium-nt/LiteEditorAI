## Зачем

Когда код всё чаще пишет агент (Claude Code, Codex, Qwen, Kimi…), а не ты сам, привычный редактор
встаёт с ног на голову: в центре уже не файл, а **разговор с агентом в терминале**. Тяжёлый IDE с
сотней панелей под это избыточен, а голый терминал неудобен — не видно, какой агент уже закончил, какой
ждёт твоего ответа, что он наменял в файлах и где вообще какой проект.

**LiteEditorAI** построен вокруг этого: главный здесь твой терминал, а просмотр кода, дерево и git живут
рядом ровно настолько, насколько нужно следить за агентом, — и прячутся одной кнопкой, когда не нужны.
Это нарочно лёгкий и тихий инструмент: открыл папку — и сразу за дело, без долгой настройки.

## Что умеет

- 🛰 **Удалённый пульт (Android) — управляй редактором с планшета или телефона.** Подключение по аккаунту
  через self-hosted релей. Пульт показывает **живой экран терминала с ПК** — следи за агентом и отвечай
  ему прямо с устройства, со своей экранной клавиатурой. **Заточен под мобильный интернет:** не гоняет
  всю историю по сети, поэтому открывается и восстанавливает связь почти мгновенно даже на слабом 4G.
  Переключение проектов и вкладок, просмотр и скачивание файлов с ПК, безопасный перезапуск редактора.
  В редакторе видно, **сколько пультов подключено** — с управлением каждым (информация об устройстве,
  местоположение по запросу, отключение доступа). Безопасность всерьёз: **одобрение устройства на ПК** —
  даже зная пароль, чужое устройство не получит доступ; плюс защита от перебора пароля, отзывные сессии
  и отзыв устройств при потере. [Как подключить →](#удалённый-пульт-android)
- 🖥 **Несколько терминалов на проект — вкладки.** У каждого проекта свои живые shell-вкладки: запусти
  агента в одной, dev-сервер в другой, разовую команду в третьей. Вкладки переименовываются, а их **имена
  сохраняются между перезапусками**.
- 🚦 **Видно состояние агента с одного взгляда:** работает (спиннер) · ждёт твоего ответа (янтарный) ·
  готов (зелёный). Плюс системные уведомления и счётчик «сколько агентов ждут» — не нужно щёлкать каждый,
  чтобы понять, кто освободился. Индикатор на карточке — агрегат по всем вкладкам проекта.
- ✅ **Задачи-промпты (TODO) и авто-очередь:** копишь следующие промпты как список задач со **статусами**
  (к выполнению · в работе · готово — выполненные не удаляются, а помечаются), **важностью** и **сортировкой**.
  Любую задачу можно отправить в терминал или в авто-очередь, которая шлёт их по одному — после каждого хода
  агента ты решаешь, слать ли следующий.
- 💬 **Чат с моделями через OpenRouter (свой ключ):** добавь свой API-ключ — рядом с проектами появится
  плашка-чат. Выбор любой модели из OpenRouter с поиском, ценой за 1M токенов и размером контекста; ответы
  **стримом** с подсветкой кода и разметкой Markdown; **несколько сессий на ключ** и настраиваемое окно
  контекста (сколько последних сообщений уходит модели). Сгенерированные изображения показываются прямо в
  чате — клик увеличивает, есть кнопка «Скачать». На плашке видно **израсходовано / лимит** ключа с кнопкой
  обновления. Для быстрого вопроса модели, когда не нужен полноценный агент в терминале. Ключи хранятся локально.
- ✍️ **Обработка текста — правка фрагментов через локального агента (без API-ключей).** Рядом с проектами
  появляется категория **«Обработка текста»**: каждая плашка — это документ (файлы лежат в
  `~/.LiteEditorAI/textproc/`), с переименовываемым заголовком и реальным именем файла под ним. Клик
  открывает текст **вместо терминала**: для Markdown — переключатели **Сырой / Оформленный / Совместно**.
  Главное — **выделите любой фрагмент**, над ним всплывёт кнопка **«AI»**: в окне можно собрать запрос из
  **системного промпта** (роль ИИ), **заготовленных** частых промптов и **своего** текста, выбрать агента —
  и фрагмент уходит на обработку. Ответ приходит куском текста на замену; прогоны **копятся версиями**
  (◀ ▶), можно **«Заменить»** выбранный или **«Попробовать ещё»**. Обработка идёт **локальным агентом
  (Claude Code или Codex) по вашей подписке — без API-ключей**. Удобно изучать и править научные статьи:
  «перескажи кратко», «упрости формулу», «исправь стиль». Системные и заготовленные промпты пополняются сами.
- 🧩 **Модули справа от терминала — один за раз.** Вспомогательные панели открываются в едином правом слоте
  (меню «Модули», и прячутся кнопкой): **Код** (вивер), **Git**, **Контейнеры**, **Базы данных**,
  **Удалённые хосты**, **Системный терминал**. Рабочий терминал остаётся главным, окно подстраивается под открытый модуль.
- 👁 **Код рядом с терминалом:** дерево файлов, подсветка, правка и сохранение, поиск, **миникарта кода**,
  дифф изменений vs HEAD, превью Markdown / картинок / HTML. Дерево само обновляется, пока агент правит файлы.
- ⎇ **Git как панель сбоку (в стиле JetBrains):** ветки и переключение, изменения с диффом по клику,
  commit / push / pull / fetch, **stash и возврат**, откат файла или всех правок, **история коммитов**
  (клик — копировать хеш) — открывается справа на выбранный проект. Плюс статус прямо в дереве файлов.
- 🐳 **Контейнеры — Docker и Podman в одной панели (замена десктопным GUI):** вкладки **Docker | Podman**,
  контейнеры сгруппированы по compose-проектам (сворачиваемые группы с сортировкой и градиентом), поды,
  образы, тома и занятое место на диске. Управление прямо из UI — старт / пауза / стоп / перезапуск /
  удаление, поштучно и **по всей группе**. Клик по контейнеру открывает **живые логи** и **встроенный
  терминал внутрь контейнера** (`exec`) — всё в окне редактора. Не установлено — так и пишет.
- 🗄 **Базы данных — лёгкий клиент Postgres / MySQL · MariaDB / SQLite (замена DataGrip для рутины):**
  подключения **по хосту или через SSH-туннель**, сгруппированы по категориям; дерево схемы, просмотр данных
  таблицы в гриде с пагинацией, **SQL-консоль** с подсветкой и выполнением по `Ctrl+Enter`, экспорт
  **CSV / JSON / SQL** и импорт SQL-файла. Пароли шифруются системным хранилищем ключей; режим **«только
  чтение»** запрещает изменяющие запросы (на стороне сервера). Драйверы встроены — ставить ничего не нужно.
- 🔌 **Удалённые хосты — серверы и хостинги прямо в редакторе:** профили подключений по **SSH / SFTP / FTP**,
  сгруппированные по категориям; вход **в один клик** и несколько живых сессий как **вкладки**. SSH — это
  полноценный терминал на сервере (вход по паролю или **по ключу из системы, без пароля**) с **keepalive**,
  чтобы соединение не рвалось. У каждого подключения — **просмотр файлов** (кнопка-папка): дерево SFTP/FTP
  с навигацией и чтением. Пароли шифруются системным хранилищем ключей и не покидают бэкенд.
- 🖳 **Системный терминал как модуль с вкладками:** отдельные шеллы вне проектов (домашняя папка) —
  несколько вкладок, для разовых системных команд рядом с рабочим терминалом.
- 🗂 **Порядок в проектах:** категории (создать, переименовать, менять порядок, свернуть; плюс «Архив» для
  отложенных), избранное, акцент-цвета, авто-скан папки с проектами.
- 🎨 **6 тем оформления:** Неоморфизм (по умолчанию), Стекло, Material, Catppuccin, Gruvbox, Aurora —
  тёмные, аккуратные, с единым акцентом. Терминал перекрашивается под тему.
- 🔔 **Уведомление об обновлениях:** редактор сам сверяется с последним релизом и подсвечивает плашку у
  номера версии, когда выходит новая — клик ведёт на страницу загрузки (само ничего не ставит).
- 🪶 **Без перегруза и кросс-платформенно:** ничего не настраивать на старте, маленькая панель настроек,
  Ctrl+C/Ctrl+V в терминале работают в любой раскладке на Linux и Windows.

## Удалённый пульт (Android)

Управляй редактором с планшета или телефона через защищённый релей — экран терминала, проекты и файлы ПК
в руке. На устройстве виден **живой экран той же сессии**, что и на ПК: можно отойти от компьютера, следить
за агентом с дивана и отвечать ему, а вернувшись — продолжить с того же места.

1. Скачай **`liteeditor-pult-*.apk`** со страницы [Releases](https://github.com/DanielLetto2020/LiteEditorAI/releases)
   и установи на Android (разреши установку из неизвестных источников).
2. В редакторе на ПК: меню **«Пульт»** → зарегистрируй аккаунт (логин/пароль).
3. В приложении на устройстве войди тем же аккаунтом.
4. Меню → **«Подключить это устройство»** → одобри на ПК (сверь код). Готово.

**Быстро на мобильном интернете.** Пульт не перекачивает всю историю терминала — он показывает текущий
экран и обновляет только то, что изменилось. Поэтому подключение и восстановление связи после провала
сети почти мгновенные, а трафик минимальный — комфортно работать даже вдали от Wi-Fi.

**Под рукой в редакторе.** Рядом с номером версии — значок с числом подключённых пультов. Клик открывает
список устройств: по каждому можно запросить информацию о нём и местоположение, или **отключить доступ**
(не удаляя — доступ возвращается одной кнопкой).

**Безопасность.** Знание пароля само по себе не даёт доступ к терминалу — устройство нужно **одобрить на
ПК**. Есть защита от перебора пароля, отзывные сессии и кнопка **«Выйти на всех устройствах»** на случай
потери планшета.

> Связь идёт через релей, который **видит трафик** (не end-to-end) — для приватного кода учитывай это.
> Пульт в стадии **alpha**.

## Установка

Готовые сборки — на странице [**Releases**](https://github.com/DanielLetto2020/LiteEditorAI/releases).

### Ubuntu / Debian (x64)
```bash
sudo apt install ./LiteEditorAI_*.deb
```
Одна команда — поставит приложение и подтянет зависимости. Запуск — иконка **LiteEditorAI** в меню приложений.

### Windows (x64)
Скачай **portable**-архив `LiteEditorAI_*-win.zip`, распакуй в любую папку и запусти **`LiteEditorAI.exe`**.
Установка не нужна. Приложение пока без цифровой подписи — SmartScreen может предупредить:
«Подробнее» → «Выполнить в любом случае».

## Сборка из исходников

```bash
npm install        # зависимости + сборка node-pty под Electron
npm start          # сборка фронта + запуск
```

Требуется Node.js 18+ (Linux/Windows x64). Для разработчиков — [CONTRIBUTING.md](CONTRIBUTING.md).

## Горячие клавиши

| Клавиши | Действие |
|---|---|
| `Ctrl+Shift+T` / `Ctrl+Shift+W` | новая / закрыть вкладку терминала |
| `Ctrl+PageUp` / `Ctrl+PageDown` | переключение вкладок терминала |
| `Ctrl+Enter` | перенос строки в терминале (продолжить ввод, не выполнять) |
| `Ctrl+C` / `Ctrl+V` | копировать выделение / вставить (в любой раскладке) |
| `Ctrl+\` | режим «один терминал» |
| `Ctrl+K` | палитра команд |
| `Ctrl+F` | поиск (в терминале или в открытом файле) |
| `Ctrl+S` | сохранить файл |
| `Ctrl+1..9` / `Ctrl+Tab` | переключение проектов |
| `Ctrl + +/−` | размер шрифта · `F11` — полный экран |
