# Ты создаёшь модуль для LiteEditor

Эта папка — пользовательский модуль редактора LiteEditor (панель в правом слоте редактора).
Твоя задача — реализовать/доработать его по запросу пользователя.

## Обязательно

1. **Сначала прочитай `GUIDE.md` в этой папке** — там полная спецификация: схема
   `manifest.json`, точные сигнатуры API `ctx` (v1), список CSS-токенов тем, правила
   и рабочий пример. Вся информация для разработки — там, никуда больше ходить не нужно.
2. **Если рядом есть `TASK.md` — это текущее задание пользователя, выполняй его.**
   Нет файла — спроси пользователя, что должен делать модуль.
3. Весь код модуля — в `index.js`: `export function activate(ctx)` и
   `export function deactivate()`. Сторонние библиотеки — только вбандленные в этот файл.
4. UI строится только внутри `ctx.panel.element`; цвета/тени/радиусы — только через
   `var(--токен)` (список в GUIDE.md); свои CSS-правила — с префиксом `.ext-<id>`.
5. `deactivate()` обязан снимать таймеры и вызывать все сохранённые функции отписки.
6. `manifest.json` уже создан — не меняй `id` и `apiVersion`; `name`/`description`/`icon`
   менять можно.
7. Не трогай файлы вне этой папки.

## Проверка результата

Сборки нет — редактор загружает `index.js` напрямую. После правок пользователь проверяет
сам: меню «Модули → (имя модуля) → Перезагрузить» (или «Пересканировать»). Когда закончишь —
скажи пользователю, что модуль готов и как его открыть.
