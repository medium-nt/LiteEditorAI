# LiteEditor Relay — свой сервер для пульта (self-hosting)

Пульт (Android) не управляет редактором напрямую — обе стороны соединяются с **релеем**
(сигнальный сервер) и он склеивает их по аккаунту. По умолчанию редактор и пульт **не привязаны**
к чьему-либо релею: вы поднимаете **свой** на своём VPS и указываете его адрес в редакторе и в пульте.

Релей — это «тупой» пересыльщик сообщений: у него своя база аккаунтов (SQLite), он не зависит ни от
какого внешнего API. Что отправляет редактор — то получает пульт, и наоборот.

> ⚠️ Релей видит трафик в открытом виде (end-to-end шифрования нет). Это **ваш** сервер — держите его
> под контролем и используйте HTTPS (в комплекте Caddy с авто-сертификатом).

## Что нужно
- VPS с Docker (или Podman) и `docker compose` / `podman compose`.
- Доменное имя, **A-запись** которого указывает на IP вашего VPS (например `relay.example.com`).
- Открытые порты **80** и **443**.

## Установка «под ключ» (один скрипт)
```bash
# 1. Скопируйте папку relay/ на свой VPS (склонируйте репозиторий или scp каталог).
cd relay

# 2. Запустите установщик — он спросит домен/email, сгенерирует секрет и поднимет всё:
bash install.sh

# 3. Через минуту проверьте (Caddy выпускает сертификат):
curl https://relay.example.com/health        # → ok
```
`install.sh` сам подберёт `docker`/`podman`, создаст `.env` (со случайным `RELAY_SECRET`) и поднимет
релей + Caddy с авто-HTTPS. Повторный запуск ничего не ломает: существующий `.env`/`./data` сохраняются.

### Вручную (если не хотите скрипт)
```bash
cp .env.example .env        # впишите RELAY_DOMAIN / RELAY_ACME_EMAIL / RELAY_SECRET (openssl rand -hex 32)
docker compose up -d --build    # или: podman compose up -d --build
```

После запуска:
- В **редакторе** (меню «Пульт») укажите хост `relay.example.com`, зарегистрируйте аккаунт.
- В **пульте** на экране входа укажите тот же хост и войдите тем же логином/паролем.

## Как подключить редактор и пульт
- **Редактор:** меню → «Пульт» → поле **«Хост релея»** = `relay.example.com` → зарегистрируйтесь
  (создаётся аккаунт на ВАШЕМ релее) → включите тумблер.
- **Пульт (APK):** на экране входа поле **«Хост релея»** = `relay.example.com` → войдите тем же
  логином/паролем. Хост запомнится.

Аккаунты у каждого релея свои — регистрируйтесь на своём. К одному аккаунту можно подключить
несколько пультов.

## Рекомендации по безопасности
- Сгенерируйте свой `RELAY_SECRET` (НЕ оставляйте значение из примера).
- Для личного сервера включите одобрение устройств: в `.env` `RELAY_REQUIRE_PAIRING=1` — тогда новый
  пульт нужно подтвердить на ПК.
- Остальные лимиты (брутфорс/регистрации/соединения) включены с безопасными значениями по умолчанию,
  см. `compose.yml` и `.env.example`.

## Обслуживание
```bash
docker compose logs -f relay     # логи
docker compose restart relay     # перезапуск (аккаунты в ./data сохраняются)
docker compose down              # остановить
```
База лежит в `./data/` (том), сертификаты — в томе `caddy_data`. Бэкап = скопировать `./data/`.

## Тесты (опционально, для разработчиков)
```bash
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt pytest
pytest -v
```

## Файлы
- `install.sh` — установщик «под ключ» (спрашивает домен/email, генерит секрет, поднимает стек).
- `relay.py` — весь сервер (FastAPI). Менять не нужно.
- `Dockerfile` / `compose.yml` / `Caddyfile` — сборка и запуск с авто-HTTPS.
- `.env.example` — шаблон настроек.
- `SECURITY_PLAN.md` — модель угроз и слои защиты (для любопытных).
- `test_relay.py` / `conftest.py` — тесты безопасности.
