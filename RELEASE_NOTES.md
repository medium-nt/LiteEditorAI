**LiteEditorAI v1.0.160** — лёгкий редактор для работы с ИИ-агентами в терминале.

Большой апгрейд **Git-модуля**. Панель разбита на три вкладки — **Изменения / История / Ветки** — а
вкладка «Изменения» стала удобным двухпанельным рабочим пространством: слева список файлов, справа сразу
виден дифф. Появились выборочный коммит, разрешение merge-конфликтов в отдельном окне и работа с ветками
в духе JetBrains.

## 🗂 Изменения — двухпанельно

Слева список изменённых файлов с **группировкой по статусу** (конфликты / изменённые / новые / удалённые /
неотслеживаемые) и **карточками-сводками** (всего, войдёт в коммит, конфликтов), справа — **превью диффа**
выбранного файла с подсветкой синтаксиса.

## ✅ Выборочный коммит

Чекбоксы у файлов — **Commit** / **Commit & Push** коммитят только отмеченное (все включены → ловит и новые
файлы). Кнопки «Все / Снять», счётчик «в коммит из N», коммит заблокирован, пока есть неразрешённые конфликты.

## 🌿 Ветки и конфликты

Вкладка **«Ветки»** в духе JetBrains: переключение, обновление **без перехода** (fast-forward upstream),
новая ветка от текущей, **merge** в текущую. **Разрешение merge-конфликтов** — отдельное окно на три панели
(моё / из ветки / результат) с подсветкой конфликтных строк.

## 🔎 История и шапка

**Поиск по истории** коммитов (хеш / сообщение / автор / дата / ссылки). Шапка панели — карточка ветки с
состоянием относительно upstream (`up to date` / `↑↓`) и последним коммитом; кнопки действий показывают
индикатор загрузки.

## 🐛 Исправлено

**WEB/SEO аудит**: рендер защищён от зависания шагов общим таймаутом — этап больше не «висит» бесконечно.

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🤖 **`liteeditor-pult-*.apk`** — пульт для Android. **В этом релизе пульт не менялся** — подойдёт APK из
  прошлого релиза; если он уже установлен, переустанавливать не нужно.

> Alpha: терминалы не переживают перезапуск редактора. Связь пульта идёт через релей, который видит трафик (не end-to-end). Баги и идеи — в [Issues](https://github.com/DanielLetto2020/LiteEditorAI/issues).
