**LiteEditorAI v1.1.50** — лёгкий редактор для работы с ИИ-агентами в терминале.

**Большая доработка самого ядра редактора: устойчивость, новые возможности терминала и гигиена кода.**
После серии апдейтов модулей этот релиз — про фундамент: терминалы, горячие клавиши, жизненный цикл окон
и фоновых процессов.

## ✨ Добавлено
- **Снимки сессий между перезапусками.** Живой процесс терминала перезапуск приложения не переживает, но
  его **вывод** — теперь да: при следующем запуске над свежим терминалом восстанавливается история
  («снимок до перезапуска»), так что видно, на чём Вы остановились. Снимок сохраняется при закрытии и
  периодически (страховка от сбоя).
- **Глобальный поиск по всем терминалам** (`Ctrl+Shift+F` или палитра команд): ищет по выводу всех
  открытых сессий сразу, показывает совпадения по сессиям; клик переводит на нужную сессию и подсвечивает
  строку. Удобно, когда параллельно работают несколько агентов.
- **Подсветка открытых модулей** в панели быстрого доступа: под кнопкой модуля с открытым окном загорается
  акцентная точка.

## 🐛 Исправлено
- **`Ctrl+\` больше не завершает работающего агента.** Раньше эта и несколько других горячих клавиш при
  фокусе в терминале срабатывали дважды — выполняли действие редактора И отправляли управляющий код в
  процесс (для `Ctrl+\` это был сигнал принудительного завершения). Теперь перехват строго одноразовый.
- **Корректные пометки Git для файлов с не-ASCII именами** (кириллица и т. п.) — отметки изменений и
  конфликтов больше не пропадают на таких файлах.
- **Окно модуля с «упавшим» содержимым теперь закрывается** обычным крестиком (раньше могло зависнуть).
- **Подсказка о ручном обновлении дерева файлов**, когда системное слежение недоступно (очень большие
  проекты / лимиты ОС) — дерево больше не «замолкает» молча.
- Поиск по терминалу использует цвета активной темы; после перерыва Помодоро фокус возвращается в терминал.

## 🔒 Безопасность и надёжность
- Внутренние служебные и секретные переменные окружения редактора больше не передаются в открываемые
  терминалы и контейнеры.
- Фоновые задачи агентов (Claude/Codex/сетевые запросы) и наблюдатели за файлами корректно завершаются при
  закрытии окон и выходе — без «осиротевших» процессов и утечек ресурсов.
- Удалённый пульт сразу переключается на новый адрес/токен релея, не дожидаясь разрыва соединения.

## 🧹 Внутреннее
- Юнит-тесты подключены к непрерывной интеграции (проверка на каждый push и pull request).
- Единая фабрика терминалов и реестр горячих клавиш — меньше дублирования, проще поддержка.
- Чистка мёртвого/неиспользуемого кода в ядре.

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** (win) — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🍎 **`.dmg`** — macOS: `-arm64` для Apple Silicon, `-x64` для Intel. Сборка без подписи Apple — при первом
  запуске снимите карантин: `xattr -dr com.apple.quarantine /Applications/LiteEditorAI.app`.
- 🤖 **`liteeditor-pult-0.60.0-debug.apk`** — пульт для Android (без изменений в этом релизе).

> Alpha: живой процесс терминала не переживает перезапуск редактора (но его вывод теперь восстанавливается
> снимком). Баги и идеи — в [Issues](https://github.com/DanielLetto2020/LiteEditorAI/issues).
