**LiteEditorAI v1.1.56** — лёгкий редактор для работы с ИИ-агентами в терминале.

**Два новых модуля — сейф паролей и мониторинг сайтов — плюс заставка-«матрица».**

## ✨ Добавлено
- **Модуль «Сейф паролей» (KeePass).** Откройте базу `.kdbx` по мастер-паролю прямо в редакторе и
  копируйте пароль / токен / логин / любое поле в буфер, не переходя в другое приложение. Поиск,
  показ секрета по кнопке, недавние базы, авто-очистка буфера через 20 секунд. Расшифровка локальная —
  мастер-пароль и пароли наружу не передаются и не сохраняются.
- **Модуль «Мониторинг сайтов».** Адреса, доступность которых редактор проверяет **в фоне** (статус +
  задержка), с историей и **уведомлением при падении/восстановлении** — мини down-detector. Аптайм,
  «проверить сейчас», баннер «недоступны N».
- **Заставка-«матрица».** Кнопка ✨ в шапке — спокойный «дождь» зелёных символов; также сама включается
  после простоя (по умолчанию 5 минут, настраивается) во всех окнах. Гасится кликом, Esc или активностью.

## 🎨 Удобства
- В окне «Проект» (вивер) в списке изменённых файлов Git — **правый клик → «Открыть файл»** (плюс
  «Показать дифф» и «Копировать путь»).

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** (win) — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🍎 **`.dmg`** — macOS: `-arm64` для Apple Silicon, `-x64` для Intel. Сборка без подписи Apple, поэтому при
  **первом** запуске разрешите её **без терминала**: запустите приложение → закройте предупреждение → откройте
  **Системные настройки → Конфиденциальность и безопасность** → пролистайте вниз и нажмите **«Открыть всё
  равно»**. Кому удобнее терминал — то же одной командой: `xattr -dr com.apple.quarantine /Applications/LiteEditorAI.app`.
- 🤖 **`liteeditor-pult-0.60.0-debug.apk`** — пульт для Android (без изменений в этом релизе).

> Alpha: живой процесс терминала не переживает перезапуск редактора (сохраняются только имена вкладок).
