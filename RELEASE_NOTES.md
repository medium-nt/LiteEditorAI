**LiteEditorAI v1.1.92** — лёгкий редактор для работы с ИИ-агентами в терминале.

**Контейнеры удалённого хоста стали «на кликах»: вместо загадочного «EOF» — точная диагностика
доступа к docker/podman-сокету и кнопка «Починить по SSH и подключиться» (редактор сам проверит
sudo, выдаст права и переподключится); контейнер с БД на сервере открывается в модуле «Базы данных»
сразу через SSH-туннель. В «Базах данных» — точечное выделение ячеек (Ctrl/Shift) с честным
копированием выбранного; в «Контейнерах» текст теперь выделяется и копируется.**

## ✨ Добавлено
- **«Починить по SSH и подключиться»**: если у SSH-пользователя нет прав на docker/podman-сокет
  сервера, а на хосте доступен sudo (без пароля — или пароль совпадает с паролем SSH-профиля),
  редактор сам выдаёт доступ и сразу переподключается. Для Docker — добавление пользователя в группу
  `docker`; для рутового Podman — точечные права на сокет через drop-in systemd (доступ получает
  только группа пользователя; контейнеры не перезапускаются). Без sudo — понятное объяснение и
  копируемая команда.
- **«Открыть в модуле БД» у контейнера удалённого хоста**: заготовка подключения приезжает сразу
  с SSH-туннелем профиля этого сервера (секреты передаются между модулями в зашифрованном виде) —
  база в контейнере сервера открывается в один клик, как локальная.
- **Точечное выделение ячеек в «Базах данных»**: **Ctrl+клик** добавляет/снимает отдельные ячейки
  (например, несколько значений одной колонки вразброс), **Shift+клик** выделяет прямоугольник от
  первой выбранной ячейки; в контекстном меню — пункт **«Копировать выбранные (N)»**.
- **Выделение текста в «Контейнерах»**: имена контейнеров и групп, строки «образ · порты», ошибки,
  версии окружения и логи можно выделить и скопировать; автообновление списка не сбрасывает
  выделение.

## 🐛 Исправлено
- **«error during connect …: EOF»** при переключении «Контейнеров» на удалённый сервер: причина —
  сокет на сервере есть, но у SSH-пользователя нет к нему прав (или sshd запрещает проброс
  unix-сокетов). Теперь редактор проверяет доступ к сокету и пробует демона через туннель ещё до
  переключения — вместо EOF показывается точная причина с командой лечения.
- SSH-туннель к сокету контейнеров **самовосстанавливается** после обрыва соединения — хост заново
  выбирать не нужно.
- Копирование выделения в таблицах БД (`Ctrl+C` и меню) переносит **именно выделенные ячейки**
  (раньше разрозненное выделение достраивалось до прямоугольника и прихватывало чужие значения);
  убрана жёлтая рамка фокуса вокруг таблицы при кликах с Shift/Ctrl.

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** (win) — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🍎 **`.dmg`** — macOS: `-arm64` для Apple Silicon, `-x64` для Intel. Сборка без подписи Apple, поэтому при
  **первом** запуске разрешите её **без терминала**: запустите приложение → закройте предупреждение → откройте
  **Системные настройки → Конфиденциальность и безопасность** → пролистайте вниз и нажмите **«Открыть всё
  равно»**. Кому удобнее терминал — то же одной командой: `xattr -dr com.apple.quarantine /Applications/LiteEditorAI.app`.
- 🤖 **`liteeditor-pult-0.60.0-debug.apk`** — пульт для Android (без изменений в этом релизе).

> Alpha: живой процесс терминала не переживает перезапуск редактора (сохраняются только имена вкладок).
