**LiteEditorAI v1.0.200** — лёгкий редактор для работы с ИИ-агентами в терминале.

Вивер кода: **git-маркеры изменений** в редакторе и надёжное обновление при правках на диске.

## ✨ Добавлено
- **Git-маркеры в гаттере вивера.** Слева от кода появилась тонкая цветная полоса, показывающая изменения
  файла относительно последнего коммита (HEAD): зелёным — добавленные строки, жёлтым — изменённые,
  красным треугольником — место, где строки были удалены. Видно сразу, не переключаясь в режим диффа.
  Обновляется при открытии, сохранении, правке файла агентом и после git-операций (коммит/checkout).

## 🐛 Исправлено
- **Конфликт при правке открытого файла.** Если файл, открытый в вивере, меняется на диске (например, его
  правит агент в терминале), а у вас есть несохранённые изменения — теперь показывается **постоянная
  плашка** с выбором «Перечитать с диска» или «Оставить мои правки» вместо исчезающего уведомления, которое
  легко было пропустить. Без несохранённых правок вивер по-прежнему подхватывает изменения с диска сам.

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** (win) — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🍎 **`.dmg`** — macOS: `-arm64` для Apple Silicon, `-x64` для Intel. Сборка без подписи Apple — при первом
  запуске снимите карантин: `xattr -dr com.apple.quarantine /Applications/LiteEditorAI.app`.
- 🤖 **`liteeditor-pult-0.60.0-debug.apk`** — пульт для Android (без изменений в этом релизе).

> Alpha: терминалы не переживают перезапуск редактора. Баги и идеи — в [Issues](https://github.com/DanielLetto2020/LiteEditorAI/issues).
