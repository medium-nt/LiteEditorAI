**LiteEditorAI v1.1.85** — лёгкий редактор для работы с ИИ-агентами в терминале.

**Большой релиз связок между модулями: сейф паролей заполняет формы подключений, «Удалённые
хосты» сканируют сервисы сервера и открывают их в модулях через SSH-туннели, «Контейнеры»
работают с docker/podman удалённого хоста, вивер выполняет `.sql` в «Базах данных», а меню
«Модули → Встроенные» стало модалкой с плитками.**

## ✨ Добавлено
- **«Сейф паролей» → формы подключений**: в формах БД / RabbitMQ / Kafka / Удалённых хостов —
  кнопки **«Из сейфа»** (логин и пароль подставляются из записи KeePass, с поиском и
  разблокировкой базы на месте) и **«В сейф»** (введённые креды сохраняются записью в `.kdbx`,
  с резервной копией базы перед записью).
- **«Удалённые хосты» → сервисы хоста**: кнопка «Сервисы» на SSH-профиле сканирует слушающие
  порты сервера и docker/podman-сокеты. PostgreSQL / MySQL / RabbitMQ / Kafka открываются в
  своих модулях **через SSH-туннель** — даже сервисы, доступные только на localhost сервера;
  веб-порты добавляются в «Мониторинг сайтов» одним кликом.
- **«Контейнеры» на удалённом хосте**: селектор «Локально / SSH-хост» — модуль подключается к
  docker/podman-сокету сервера через SSH-туннель; список, действия, логи и exec-терминал
  работают как с локальными контейнерами.
- **«Контейнеры» → «Мониторинг сайтов»**: у контейнеров с веб-интерфейсом — кнопка
  «Наблюдать»: сайт попадает в мониторинг с проверками и уведомлениями (без дублей).
- **Вивер ↔ «Базы данных»**: на `.sql`-файле — кнопка «Выполнить в БД» (выделение или весь
  файл уходит в новую SQL-консоль выбранного подключения); в экспорте результата запроса —
  кнопка **«В вивер»** (результат открывается файлом CSV/JSON/Markdown/…).
- **Файлы контейнера в вивере**: вкладка «Файлы» в карточке контейнера — обзор ФС, клик по
  файлу открывает копию в вивере (удобно глянуть конфиг внутри).
- **Правка удалённых файлов в вивере**: файл с сервера открывается из файл-браузера
  «Удалённых хостов», каждое сохранение (`Ctrl+S` / автосейв) **заливается обратно** по SFTP/FTP.
- **Меню «Модули → Встроенные» — модалка с плитками**: 20 модулей карточками с поиском и
  секциями «Следуют за проектом» / «Самостоятельные».

## 🔄 Изменено
- Повторное открытие сервиса из «Контейнеров»/«Удалённых хостов» тихо обновляет адрес
  сохранённого подключения (порт контейнера или SSH-туннеля мог смениться).

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** (win) — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🍎 **`.dmg`** — macOS: `-arm64` для Apple Silicon, `-x64` для Intel. Сборка без подписи Apple, поэтому при
  **первом** запуске разрешите её **без терминала**: запустите приложение → закройте предупреждение → откройте
  **Системные настройки → Конфиденциальность и безопасность** → пролистайте вниз и нажмите **«Открыть всё
  равно»**. Кому удобнее терминал — то же одной командой: `xattr -dr com.apple.quarantine /Applications/LiteEditorAI.app`.
- 🤖 **`liteeditor-pult-0.60.0-debug.apk`** — пульт для Android (без изменений в этом релизе).

> Alpha: живой процесс терминала не переживает перезапуск редактора (сохраняются только имена вкладок).
> Баги и идеи — в [Issues](https://github.com/DanielLetto2020/LiteEditorAI/issues).
