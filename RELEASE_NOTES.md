**LiteEditorAI v1.0.177** — лёгкий редактор для работы с ИИ-агентами в терминале.

Новый модуль **IterFlow** для работы с задачами из таск-трекера, доработки **удалённого пульта** и публикация
кода **релея** для самостоятельного хостинга.

## 📋 Новый модуль — IterFlow

Если вы ведёте проекты в таск-трекере **IterFlow** ([iter-flow.ru](https://iter-flow.ru)) — теперь его задачи
доступны **прямо в редакторе**, рядом с терминалом и агентом. Вход для исполнителя → ступенчатый выбор
**профиль → заказчик → проект**, дальше четыре вкладки:

- **Итерация** — вертикальный таймлайн итераций с датами и временем, задачи внутри; карточки сворачиваются,
  сортировка по стадии (в работе — сверху).
- **Канбан** — плоская доска задач выбранной итерации со сменой статуса.
- **Туду** — общие заметки проекта.
- **Чат** — общий канал проекта.

Задачи показываются компактной строкой-заголовком с **подсветкой по важности** и значком, если есть подробности;
клик раскрывает детали. Удобно фрилансерам и небольшим студиям, которые согласуют работу с заказчиком и хотят
брать задачи в работу с ИИ-агентом, не переключаясь в браузер.

## 🛰 Удалённый пульт

- **Свой релей.** Код сигнального релея теперь открыт (каталог `relay/` в репозитории): можно поднять
  собственный релей (Docker + Caddy, инструкция внутри) и указать его адрес в настройках — адрес больше не зашит
  в код.
- **Стабильный ID устройства.** После обновления APK пульта повторный пайринг по коду больше не нужен —
  редактор узнаёт устройство.

### ⬇️ Что скачать
- 🐧 **`.deb`** — Ubuntu / Debian (x64): `sudo apt install ./LiteEditorAI_*.deb`, затем иконка **LiteEditorAI** в меню.
- 🪟 **`.zip`** — Windows (x64), portable: распакуйте и запустите `LiteEditorAI.exe` (установка не нужна).
- 🤖 **`liteeditor-pult-*.apk`** — пульт для Android (обновлён: стабильный ID устройства).

> Alpha: терминалы не переживают перезапуск редактора. Модуль IterFlow требует аккаунта в сервисе IterFlow. Связь пульта идёт через релей (свой или сторонний), который видит трафик (не end-to-end). Баги и идеи — в [Issues](https://github.com/DanielLetto2020/LiteEditorAI/issues).
