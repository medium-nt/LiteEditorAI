# Changelog

Все заметные изменения LiteEditorAI. Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
нумерация — [SemVer](https://semver.org/lang/ru/). Проект в стадии **alpha**.

## [1.0.147] — 2026-06-13

Релиз про **безопасность и движок**: закрыты все известные уязвимости зависимостей, в том числе
обновление Electron на актуальную поддерживаемую версию. Пользовательских изменений интерфейса нет —
приложение то же, но на свежем и безопасном фундаменте.

### 🔒 Безопасность
- **Electron 33 → 42.4.0** (Chromium 138+, Node 24). Версия 33 уже не получала обновлений безопасности —
  переход на поддерживаемую ветку закрывает накопившиеся уязвимости движка и приносит свежий Chromium.
- **Закрыты уязвимости сборочных зависимостей** (`esbuild`, `@electron/rebuild` и его `tar`/`node-gyp`/
  `cacache`/`make-fetch-happen`, `tmp`) — это инструменты сборки, в дистрибутив они не входят, но репозиторий
  теперь чист. Итог `npm audit`: **0 уязвимостей** (было 8 высокого уровня).

### 🐛 Исправлено
- **Иконка приложения в доке Linux.** На новом движке окно иначе сообщает своё имя системе (`WM_CLASS`),
  из-за чего GNOME показывал отдельную плашку с обобщённой «шестерёнкой». Ярлык, тема иконок и сборка `.deb`
  приведены в соответствие — окно снова под нормальной иконкой и группируется со своим ярлыком.

### 🔧 Внутреннее
- `cpu-features` (необязательная оптимизация ssh2, несовместимая с V8 в Node 24) исключена из пересборки —
  SSH работает без неё; сборка node-pty переведена на точечный режим. CI собирает на Node 22.

## [1.0.144] — 2026-06-13

Релиз про **задачи**: список TODO стал полноценным модулем справа от терминала — с вкладками «проект» и
«общие», переносом между ними и экспортом/импортом в JSON. Заодно — доводка интерфейса: чистая карточка
проекта, разделители в панели быстрого доступа и сгруппированное меню «Модули».

### ✨ Добавлено
- **Задачи — модуль правого слота.** Список TODO переехал из всплывающего окна карточки в отдельную панель
  справа (открывается из «Модули», палитры `Ctrl+K` и квикбара), как Git или «Контекст», и следует за активным
  проектом. Интерфейс переосмыслен: читаемые карточки (статус, важность, текст во всю ширину), фильтр
  **Активные / Все / Готово** со счётчиками, новая задача — через окно с полноценным полем-описанием.
- **Две вкладки: «Проект» и «Общие».** У каждого проекта свой список задач, плюс отдельный список **общих**
  задач вне проектов. Выбранная вкладка запоминается между запусками.
- **Перенос задач между списками** — кнопки «В общие» (из проекта) и «В проект» (из общих, в активный проект).
- **Экспорт и импорт задач в JSON.** Выгрузить можно текущий список или **все задачи сразу** (по проектам +
  общие, с разделением). При импорте — выбор режима: **заменить** (по умолчанию) или **добавить** к существующим.
- **Разделители в панели быстрого доступа.** В «Модули → Настройка панели» можно добавлять вертикальные
  разделители (сколько угодно, в любое место) и группировать кнопки на полосе под терминалом.

### ♻️ Изменено
- **Карточка проекта — чище.** Кнопки вивера и Git убраны с карточки (они есть в панели быстрого доступа под
  терминалом), скрывающаяся по наведению полоса действий тоже убрана — остались только ★ «избранное» и ⋮ «меню».
- **Меню «Модули» — группировка.** Пункты стали двухстрочными (название + описание помельче), а длинный список
  свёрнут в раскрывающиеся подменю **«Встроенные»** и **«Мои модули»** + отдельный пункт «Настройка панели».
- Панель задач по умолчанию шире — длинные формулировки читаются целиком.

### 🗑 Удалено
- **Авто-очередь заметок.** Полуавтоматическая отправка задач в агента «по одной» убрана как неудобная;
  задачи по-прежнему отправляются в терминал вручную (с удалением или без).
- Пункт **«Git»** из контекстного меню карточки проекта (дублировал панель быстрого доступа).

## [1.0.143] — 2026-06-12

Два больших новшества: модуль **«Контекст»** — визуальный конструктор контекста агента (CLAUDE.md /
AGENTS.md как граф на канве), и **цветной пульт** — экран терминала на Android-пульте теперь с полной
палитрой, как на ПК.

### ✨ Добавлено
- **Модуль «Контекст» — граф контекста агента.** Канва в духе n8n: блоки-источники (текст, файл, команда,
  слот памяти агента) соединяются проводами с выходами **Claude (CLAUDE.md)** и **Codex (AGENTS.md)**;
  блок через **группу-профиль** — переключаемая часть контекста (тумблер включает/выключает целиком).
  Кнопка «Подтвердить» собирает файлы; перед перезаписью существующих делается **бекап** (папка и глубина
  хранения настраиваются), чужой файл без подтверждения не перезаписывается. Счётчики ≈токенов на блоках
  и выходах показывают «вес» контекста.
- **«Контекст»: распил готового CLAUDE.md.** Существующий CLAUDE.md или AGENTS.md можно одной кнопкой
  разобрать на блоки **вашим локальным агентом** (Claude Code / Codex — по вашей подписке, без API-ключей):
  агент разбивает файл на смысловые части, и они появляются на канве уже соединёнными. Исходник
  предварительно бекапится.
- **«Контекст»: профили.** У проекта может быть **несколько графов-профилей** (например, «разработка» и
  «ревью») — вкладки над канвой: создать, переименовать, удалить, переключить; видно, какой профиль
  сейчас применён, а несобранные изменения отмечаются индикатором.
- **«Контекст»: библиотека блоков** — общие блоки, переиспользуемые между проектами; **справка «?»** —
  как всё устроено, простым языком; **markdown-превью** содержимого блоков.
- **Пульт: цветной экран терминала (APK 0.54.0).** Экран на пульте теперь раскрашен как настоящий
  терминал: полная палитра (16 / 256 / RGB), фон, жирный/курсив/подчёркивание/инверсия — диффы агента
  зелёные и красные, а не серые. Трафик почти не вырос: цвета едут внутри тех же экономных кадров-диффов.
- **Пульт: курсор больше не сползает** на строках с эмодзи или иероглифами левее него.
- **Квикбар — панель быстрого доступа к модулям.** Меню **«Модули → Настройка панели»**: выбранные модули
  (включая ваши собственные) становятся кнопками-иконками в полосе под терминалом — один клик вместо
  похода в меню. Состав и порядок настраиваются и сохраняются.

### 🔧 Изменено
- Удалён мёртвый код старой схемы размеров пульта (наследие до «проекции экрана») — меньше кода, меньше
  сюрпризов.

### 🐛 Исправлено
- **Ctrl+F-поиск в терминале** не открывался на вкладках-сессиях (искал по идентификатору проекта вместо
  сессии).
- **Вложенные модалки** не ломали друг друга при закрытии (закрытие дочерней сносило обработчики
  родительской).
- **Утечка обработчиков** в списке «Пульты» при закрытии окна по Esc или клику в фон.
- **«Контекст»: распил падал** на CLAUDE.md с код-блоками (агент возвращал JSON, который ломался о
  ```-разметку внутри содержимого) — теперь разбирается корректно, а при настоящей ошибке агента
  показывается его сырой ответ с кнопками «копировать/повторить» вместо глухого тоста.

### 🔒 Безопасность
- Markdown-превью блоков «Контекста» рендерится с санацией HTML (скрипты, iframe, on*-атрибуты и
  javascript:-ссылки вырезаются) — содержимое блока может прийти из чужого файла при распиле.

## [1.0.135] — 2026-06-11

Главное — **пользовательские модули (плагины)**: теперь редактор можно расширять своими панелями,
которые пишет ваш ИИ-агент.

### ✨ Добавлено
- **Свои модули (плагины).** Меню «Модули → Создать модуль…» открывает менеджер пользовательских модулей.
  Вы задаёте имя — редактор создаёт заготовку в `~/.LiteEditorAI/modules/<имя>/` и **открывает терминал
  прямо в её папке**, где код модуля пишет **любой ваш локальный ИИ-агент** (Claude Code, Codex, …) по
  встроенной спецификации. Модуль — это ES-модуль с одним файлом `index.js` (`activate(ctx)`/`deactivate()`),
  получающий узкий API `ctx` v1 (панель, UI-хелперы редактора, хранилище, проекты, команды палитры, тема).
- **Менеджер модулей.** Готовые модули появляются справа рядом со встроенными (Git, Базы данных и т.п.),
  открываются из меню «Модули → Мои модули», **перезагружаются на лету** после правок (кнопка ⟳), у каждого
  есть «открыть папку» и удаление (через корзину ОС).
- **Каталог `module-kit/` для авторов.** Самодостаточная спецификация `GUIDE.md` (схема манифеста, сигнатуры
  `ctx`, список токенов тем, правила), рабочий пример-калькулятор и подсказки для ИИ-агентов — всё рядом с
  каждым создаваемым модулем, чтобы агент писал модуль без дополнительных вводных.

### 🐛 Исправлено
- Иконки в модулях и менеджере больше не показываются с белым фоном — используют нативный стиль кнопок темы.

## [1.0.128] — 2026-06-11

Большое обновление **удалённого пульта (Android)**: он стал быстрым на мобильном интернете и удобнее в
работе. Плюс — рефакторинг редактора «под капотом» и набор исправлений.

### ✨ Добавлено
- **Пульт открывается и восстанавливает связь почти мгновенно — даже на слабом мобильном интернете.**
  Раньше при каждом подключении и после любого провала сети пульт перекачивал всю историю терминала
  (сотни килобайт — единицы мегабайт) и заново её перерисовывал — на 4G это были долгие загрузки и
  бесконечные «переподключения». Теперь пульт показывает **текущий экран терминала** и обновляет только
  то, что изменилось: восстановление связи стоит ~5 КБ вместо мегабайтов, трафик упал **на 1–2 порядка**.
  Работать вдали от Wi-Fi стало комфортно.
- **Пульт: вертикальная компоновка под телефон/планшет.** Экран зафиксирован вертикально (не нужно вертеть
  устройство), список проектов и терминалов открывается **шторкой сверху** (не отнимает ширину у
  терминала), а меню — аккуратным выпадающим списком. У каждого проекта всегда есть вкладка терминала —
  одно касание открывает её на ПК.
- **Пульт: история по запросу.** Прокрутка вверх **подгружает прошлый вывод** частями (как лента в
  мессенджере), а возврат вниз освобождает память. Поле ввода агента всегда на виду и подсвечено.
- **В редакторе видно подключённые пульты.** Рядом с номером версии — значок **📱 N**. Клик открывает
  список устройств: по каждому можно запросить **информацию об устройстве** и **местоположение**, а также
  **отключить доступ** — не удаляя устройство (доступ возвращается одной кнопкой).

### 🔧 Изменено
- **Редактор переразложен на модули «под капотом».** Большие части интерфейса (Git, Контейнеры, Базы
  данных, Удалённые хосты, OpenRouter, Задачи, Обработка текста) вынесены в отдельные изолированные модули,
  а общие UI-хелперы — в один файл. Для пользователя поведение не меняется; цель — чище код и меньше
  случайных поломок при доработках. Заодно удалены легаси-зависимости и мёртвый код.
- **Системный терминал — только как модуль.** Дублирующая иконка из шапки убрана; открывается через меню
  **«Модули» → «Системный терминал»**.

### 🐛 Исправлено
- **Пункты меню «Файл / Настройки / Модули» и «Управление ключами» OpenRouter** падали с ошибкой
  (последствия рефакторинга) — исправлено.
- **Модуль «Удалённые хосты»** не открывал список при категориях — исправлено.
- **Окно настроек** могло вылезать за верх и низ экрана на невысоких дисплеях — теперь вписывается и
  прокручивается внутри.
- **Пульт:** убраны лишние полосы-разделители над строкой ввода (TUI-агенты вроде Claude Code); приветствие
  свежего терминала больше не «прячется» за верхним краем; прокрутка истории вверх показывает текст, а не
  повторяющиеся символы.

## [1.0.109] — 2026-06-05

### ✨ Добавлено
- **«Обработка текста» — новая категория-интеграция: правка фрагментов через локального агента.** Рядом с
  проектами (ниже «OpenRouter») появилась категория **«Обработка текста»**.
  - **Документы-плашки.** Каждая плашка — документ; файлы хранятся в `~/.LiteEditorAI/textproc/`. У плашки —
    **переименовываемый заголовок** и под ним **реальное имя файла** (менее заметной строкой). Создание
    кнопкой **＋**, удаление — в корзину.
  - **Текст вместо терминала.** Клик по плашке открывает документ в центральной зоне (как чат OpenRouter).
    Переключатель **«Текст / MD»**; для Markdown — **«Сырой / Оформленный / Совместно»** (исходник, превью,
    или две половины рядом с живым обновлением). Автосохранение.
  - **Главное — кнопка «AI» над выделением.** Выделите любой фрагмент в исходнике → всплывает кнопка **«AI»**.
    В окне сверху вниз: сам **выделенный фрагмент**, выбор **агента** (Claude / Codex), **системный промпт**
    (роль ИИ), **заготовленные промпты** (клик подставляет в поле) и **своя текстарея** — обработать можно,
    выбрав хотя бы одно. Ответ приходит **куском текста на замену**; прогоны **копятся версиями** (◀ ▶) —
    **«Заменить»** применяет выбранный, **«Попробовать ещё»** генерирует новый без потери прошлых.
  - **Без API-ключей — по подписке.** Обработка идёт **локальным агентом** (`claude -p` или `codex exec`),
    то есть вашей подпиской Claude Code / Codex, а не платным API-ключом.
  - **Библиотека промптов.** Системные и заготовленные промпты создаются и редактируются (⚙ в шапке категории),
    агент по умолчанию настраивается там же.

### 🔒 Безопасность
- **Markdown-превью документов рендерится без «сырого» HTML** (raw-HTML выкидывается на этапе рендера) —
  документ, вставленный из внешнего источника, не может протащить разметку с инъекцией. Поверх и так
  действует строгий CSP (`script-src 'self'`).

## [1.0.108] — 2026-06-04

### 🔧 Изменено
- **«Заметки» проекта переделаны в TODO-список со статусами.** Вместо карточек — компактные строки задач:
  - **Статус** (клик по чекбоксу слева циклит): ☐ к выполнению → ◐ в работе → ☑ выполнено. **Выполненные
    задачи не удаляются** — просто помечаются (зачёркнуты и приглушены); отдельная корзина (с подтверждением) —
    для реального удаления.
  - **Важность** задачи (флажок): обычная → важная → срочная, с цветной полосой слева.
  - **Сортировка** (запоминается): вручную (перетаскивание / ▲▼), по важности, по статусу.
  - Кнопки действий — иконками в стиле карточек проекта, всегда на виду. Отправка в терминал и **авто-очередь**
    работают как раньше. Бейдж на карточке проекта теперь считает **активные** (не выполненные) задачи.
  - **Данные сохранены:** старые заметки открываются как задачи «к выполнению» — формат хранения совместим,
    ничего не теряется.

## [1.0.107] — 2026-06-04

### ✨ Добавлено
- **Модуль «Удалённые хосты» (RemoteHost) — серверы и хостинги прямо в редакторе.** Открывается из меню «Модули».
  - **Профили подключений по `SSH` / `SFTP` / `FTP`**, сгруппированные по категориям; создание / правка / тест,
    вход **в один клик**.
  - **SSH — полноценный терминал на сервере** (живой shell), несколько сессий как **вкладки** с переключением
    (`Ctrl+PageUp/PageDown`, `Ctrl+Shift+W`). Вход **по паролю** или **по ключу из системы без пароля**
    (SSH-агент / `~/.ssh`, как `ssh user@host`). **Keepalive** с настраиваемым интервалом — чтобы соединение
    не рвалось при простое.
  - **Просмотр файлов** (кнопка-папка у подключения): дерево удалённых файлов по **SFTP** (для SSH/SFTP) или
    **FTP/FTPS** с навигацией, размерами и чтением содержимого (пока только для просмотра).
- **Миникарта кода в вивере (как в VSCode):** уменьшенная копия файла справа от кода с рамкой текущей области
  и кликом-прыжком. Включается/выключается кнопкой в шапке вивера, состояние запоминается.
- **Пункт «Вивер» в меню «Модули»** — открыть просмотр файлов активного проекта (как кнопка-«глаз» на карточке);
  при открытой категории OpenRouter показывает заглушку «нужно выбрать проект».

### 🔧 Изменено
