# Changelog

Все заметные изменения LiteEditorAI. Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
нумерация — [SemVer](https://semver.org/lang/ru/). Проект в стадии **alpha**.

## [1.0.167] — 2026-06-15

Новый слой работы с **ошибками и логами**, полировка **Git-модуля** и открытый **приём pull request'ов**
от сообщества.

### ✨ Добавлено
- **Реестр ошибок** — новая вкладка **«Ошибки»** в окне «Логи»: предупреждения и ошибки редактора
  схлопываются по сигнатуре в записи со **статусом** (открыта / решено / игнор), счётчиком повторов и
  временем появления — видно, что реально сломано, без чтения сырых логов. Статус можно отметить вручную
  или передать список ошибок агенту на разбор.
- **Управление логами из интерфейса**: удаление отдельного лог-файла и очистка старых; суммарный потолок
  каталога логов (30 МБ) и автоматический перепрунинг по таймеру — каталог логов не разрастается.
- **Git-модуль**: **чип состояния ветки** относительно upstream (↑ впереди / ↓ отстаёт / «upstream удалён»)
  в шапке панели и во вкладке «Ветки»; список веток открывается аккуратной выпадашкой, которая больше
  **не уезжает за правый край экрана**.
- Открыт **приём pull request'ов** через публичную ветку **`contrib`** — форкните репозиторий и пришлите
  PR в `contrib` (см. [CONTRIBUTING.md](CONTRIBUTING.md)).

### 🐛 Исправлено
- **Git-модуль**: если коммит прошёл, а push — нет, список изменений теперь корректно обновляется (файлы
  уже в коммите) и показывается ошибка push, а не прежнее «зависшее» состояние.
- Совместимость с новой сигнатурой события `console-message` в свежем Electron (теперь приходит
  Event-объект) — проброшенные из рендерера сообщения снова попадают в лог.

## [1.0.160] — 2026-06-14

Большой апгрейд **Git-модуля**. Панель разбита на три вкладки — **Изменения / История / Ветки** —
а вкладка «Изменения» переработана в удобное двухпанельное рабочее пространство: слева список файлов,
справа сразу виден дифф. Появились **выборочный коммит**, **разрешение merge-конфликтов** в отдельном
окне и работа с ветками в духе JetBrains.

### ✨ Добавлено
- **Три вкладки** в Git-панели: **Изменения**, **История**, **Ветки**.
- **Вкладка «Изменения» — двухпанельный вид**: слева список изменённых файлов с **группировкой по
  статусу** (конфликты / изменённые / новые / удалённые / неотслеживаемые) и **карточками-сводками**
  (сколько всего, сколько войдёт в коммит, сколько конфликтов), справа — **превью диффа** выбранного
  файла с подсветкой синтаксиса.
- **Выборочный коммит** — чекбоксы у файлов; **Commit** / **Commit & Push** коммитят только отмеченное
  (все включены → ловит и новые файлы). Кнопки «Все / Снять», счётчик «в коммит из N», блокировка
  коммита, пока есть неразрешённые конфликты.
- **Вкладка «Ветки»** в духе JetBrains: на каждую ветку — переключение, обновление **без перехода**
  (fast-forward upstream), новая ветка от текущей, **merge** в текущую; текущая ветка с бейджем сверху.
- **Разрешение merge-конфликтов** — отдельное окно на **три панели** (моё / из ветки / результат) с
  подсветкой конфликтных строк; на вкладке «Изменения» — бейдж со счётчиком конфликтов.
- **Поиск по истории** коммитов (фильтр по хешу / сообщению / автору / дате / ссылкам), refs показаны
  отдельными чипами.
- Шапка панели — **карточка ветки** с состоянием относительно upstream (`up to date` / `↑↓`) и последним
  коммитом; кнопки действий показывают индикатор загрузки во время операции.
- **Discard all** перенесён в блок **Sync** рядом с fetch / pull / push / stash.
- **Подсветка синтаксиса в диффе** — `renderDiffInto` учитывает имя файла; изолированная фабрика
  `createCodeEditor` (CodeMirror с темой вивера) переиспользуется и в окне разрешения конфликтов.

### 🐛 Исправлено
- **WEB/SEO аудит**: рендер обёрнут защитой от зависания шагов (DOM / скриншоты / CDP-эмуляция) с общим
  таймаутом — этап больше не «висит» бесконечно. Убрана кнопка «Передать агенту» (модуль самостоятельный,
  не привязан к проекту).

## [1.0.157] — 2026-06-14

Новый модуль **«WEB/SEO аудит»** — самостоятельный анализатор сайтов: проверяет и локальный
dev-сервер, и внешний домен, ведёт свой список сайтов с историей аудитов, а находки можно одной
кнопкой передать локальному агенту для исправления. Встроенный Chromium рендерит страницу
по-настоящему — поэтому видны и метрики скорости, и SEO современных SPA-сайтов.

### ✨ Добавлено
- **Модуль «WEB/SEO аудит»** — отдельная панель (не привязана к проекту): свой **список сайтов**, у
  каждого — **история аудитов** с дельтой и трендом оценок. Локальные dev-серверы находятся автопоиском
  по типовым портам. Результаты приходят **поэтапно**: быстрый скан виден сразу, скриншоты/метрики и
  проверка ссылок досыпаются фоном.
- **Безопасность** — заголовки (CSP / HSTS / X-Frame-Options / X-Content-Type-Options / Referrer-Policy /
  Permissions-Policy) с оценкой и советом, **TLS-сертификат** (издатель, срок, шифр, цепочка, доверие),
  куки (Secure / HttpOnly / SameSite), обнаружение **открытых `.git/` и `.env`**, утечки версий ПО,
  mixed content.
- **SEO** — title / description / canonical / viewport / lang, OpenGraph и JSON-LD, структура заголовков,
  картинки без `alt` и размеров — **из отрендеренной страницы** (скрытый Chromium видит контент,
  добавленный JavaScript); проверка **битых ссылок**; robots.txt / sitemap.xml / security.txt.
- **Скорость** — **Core Web Vitals** (LCP, CLS, FCP, TTFB) и тайминги, **вес страницы** и число запросов
  с разбивкой по типам, тяжёлые ресурсы, сторонние запросы, ошибки в консоли; **скриншоты** десктоп и
  мобильный.
- **Сеть/Домен** — цепочка редиректов, DNS-записи, почтовая гигиена **SPF / DMARC**, **WHOIS** (возраст
  домена, регистратор, срок), **гео-IP** и хостер.
- **Оценки** Безопасность / SEO / Скорость (0–100) на «Обзоре», сводка в буфер и **экспорт отчёта**
  `.md` / `.json`.
- **Передать агенту** — все находки или одна конкретная (по кнопке) уходят в активный терминал как
  задача на исправление.

## [1.0.155] — 2026-06-14

Новый модуль **«Аудит»** — быстрый рентген любого проекта: состав, крупнейшие файлы, гигиена,
техдолг и история. Полезен и человеку (понять незнакомый проект, найти раздувание и мусор), и
агенту (сводка-паспорт одной кнопкой в буфер).

### ✨ Добавлено
- **Модуль «Аудит»** — панель правого слота, следует за активным проектом. Источник файлов на выбор:
  **git-tracked** (честный фильтр — без `node_modules` и сборки) или весь рабочий каталог. Сканирование
  делает один проход по дереву; крупные деревья и бинарь обрабатываются с предохранителями.
- **Обзор** — паспорт проекта (файлы · строки · вес, языки по строкам, категории) и **чипы-находки** со
  счётчиками; кнопки **«Скопировать сводку»** (markdown-паспорт в буфер — для агента или заметки) и
  **«Экспорт отчёта»** (полный отчёт в `.md` / `.json`).
- **Типы** — сортируемая таблица по расширениям (файлов · строк · вес · доля); клик по типу раскрывает
  список его файлов, переход по категории — из «Обзора».
- **Крупные** — топ файлов **по строкам / по весу** с флагом **«аномалия»** (файл сильно выше нормы для
  своего типа) и фильтром «только аномалии».
- **Медиа** — картинки, видео, архивы и шрифты по весу.
- **Гигиена** — мусор в гите (каталоги сборки, `.env`, временные/служебные файлы), минифицированные и
  длинные строки, **дубликаты по содержимому**, возможно осиротевшие файлы (эвристика, с пометкой).
- **Долг** — метки **TODO / FIXME / HACK** и **потенциальные секреты** (ключи/токены/приватные ключи);
  клик по находке открывает файл на нужной строке.
- **История** — **горячие файлы** по git-churn (где больше всего правок), свежие изменения и давно не
  тронутые файлы (на не-git проектах — по времени файла).
- **Действия по файлу** — в любом списке: открыть в вивере (клик по строке), копировать путь, отправить
  в терминал запрос на разбор файла.

## [1.0.152] — 2026-06-14

Модуль **«Контекст»** переработан целиком: распил стал точным и без потерь, собранный файл — чистым,
появились группы из заголовков, восстановление версий и отслеживание правок файла вне модуля
(агентом или вручную).

### ✨ Добавлено
- **Распил по заголовкам — детерминированный, без потерь.** Любой markdown-заголовок (`#`…`######`) становится
  отдельным блоком, а обратная сборка даёт исходный файл без искажений (проверено побайтно). Выполняется
  мгновенно, без обращения к ИИ.
- **Чистый файл контекста.** Собранные `CLAUDE.md` / `AGENTS.md` больше не содержат служебных
  комментариев-маркеров — это обычный markdown. Принадлежность файла модулю определяется по контрольной сумме.
- **Группы из заголовков.** Заголовок-контейнер (без своего текста, но с вложенными под-заголовками) при распиле
  автоматически становится **группой** (вложенность любой глубины). Два вида групп: «обёртка» (только
  переключатель) и «группа-текст» (пишет свой заголовок в файл). Ручная группировка выделенных блоков — `Ctrl+G`.
- **Восстановление версий.** Любую версию из «🕘 Версии» можно вернуть на канву одним блоком или назначить
  **новым «Оригиналом»** (базовый снимок обновляется, прежний становится обычной версией).
- **Отслеживание внешних правок.** Если файл изменён вне модуля (например, агентом), в шапке загорается значок,
  а окно сверки показывает новые / изменённые / удалённые разделы. Для конфликтов — выбор **«Оставить моё /
  Взять из файла / Соединить»**. Сборка больше не затирает внешние правки молча.
- **Экспорт профиля.** Пункт «Сохранить полный файл…» сохраняет собранный файл в любое место на диске.
- **Канва.** Выделение рамкой (`Shift`+протягивание) и групповое перемещение, удаление провода по наведению.

### 🔧 Изменено
- **Профили** — одна строка: имя + значок активного (собранного в файл) профиля, действия в меню «⋯»
  (переименовать, дублировать, сохранить файл, удалить). Активный профиль удалить нельзя.
- **Заголовок текстового блока** берётся из его содержимого автоматически и не редактируется вручную.
- **«Подтвердить» / «Сбросить»** — компактные кнопки-иконки; распил доступен прямо на блоке (синий значок ✂,
  если в блоке несколько заголовков). Выходной блок больше нельзя выключить.

## [1.0.151] — 2026-06-13

Большая переработка модуля **«Контекст»** (сборка `CLAUDE.md` / `AGENTS.md` из блоков). Claude и Codex
теперь полностью независимы, появилась история версий файла, а сборка стала проще и нагляднее.

### ✨ Добавлено
- **Независимые агенты Claude и Codex.** Переключатель сверху: у каждого агента свои профили, своя история
  версий и свой собранный файл — можно держать, например, Claude под код, а Codex под вёрстку и применять их
  раздельно.
- **Точки восстановления (версии файла)** — «git для одного файла»: снимок **«Оригинал»** при первом открытии
  (защищён от удаления) и версия на каждую сборку. От любой версии можно **«Распилить»** — разложить её на
  блоки канвы. Так можно экспериментировать сколько угодно, оригинал всегда сохранён.
- **Стартовый блок сразу с содержимым.** При открытии профиль уже содержит текстовый блок с полным текущим
  файлом агента (или пустой, если файла нет), привязанный к выходу — можно сразу править или распилить.
- **Лёгкий онбординг и навигация по канве:** подсказка-плашка (закрывается один раз), кнопка **«уместить всё
  в экран»**, авто-центрирование блоков на старте.

### 🐛 Улучшено
- **Распил берёт версию, а не живой файл** — оригинал можно распиливать повторно даже после сборки.
- Управление вынесено на канву (правый верхний угол); в окне блока — **стрелки-листание** между текстовыми
  блоками, кнопка **«Сохранить»** появляется только при изменениях, кнопки внизу окна компактнее.
- **Удаление провода** — наведите на линию и нажмите «×» (как в n8n).

### 🔧 Изменено
- Набор блоков временно сведён к **«Текст»** и **«Группа»** (файл / команда / память агента вернутся позже).
- Хранилище модуля переработано под независимых агентов — **прежние графы «Контекста» не переносятся**;
  соберите заново (быстро через ✂ «Распилить»). Файлы `CLAUDE.md` / `AGENTS.md` при этом не затрагиваются.

## [1.0.147] — 2026-06-13

Релиз про **безопасность и движок**: закрыты все известные уязвимости зависимостей, в том числе
обновление Electron на актуальную поддерживаемую версию. Пользовательских изменений интерфейса нет —
приложение то же, но на свежем и безопасном фундаменте.

### 🔒 Безопасность
- **Electron 33 → 42.4.0** (Chromium 138+, Node 24). Версия 33 уже не получала обновлений безопасности —
  переход на поддерживаемую ветку закрывает накопившиеся уязвимости движка и приносит свежий Chromium.
- **Закрыты уязвимости сборочных зависимостей** (`esbuild`, `@electron/rebuild` и его `tar`/`node-gyp`/
  `cacache`/`make-fetch-happen`, `tmp`) — это инструменты сборки, в дистрибутив они не входят, но репозиторий
  теперь чист. Итог `npm audit`: **0 уязвимостей** (было 8 высокого уровня).

### 🐛 Исправлено
- **Иконка приложения в доке Linux.** На новом движке окно иначе сообщает своё имя системе (`WM_CLASS`),
  из-за чего GNOME показывал отдельную плашку с обобщённой «шестерёнкой». Ярлык, тема иконок и сборка `.deb`
  приведены в соответствие — окно снова под нормальной иконкой и группируется со своим ярлыком.

### 🔧 Внутреннее
- `cpu-features` (необязательная оптимизация ssh2, несовместимая с V8 в Node 24) исключена из пересборки —
  SSH работает без неё; сборка node-pty переведена на точечный режим. CI собирает на Node 22.

## [1.0.144] — 2026-06-13

Релиз про **задачи**: список TODO стал полноценным модулем справа от терминала — с вкладками «проект» и
«общие», переносом между ними и экспортом/импортом в JSON. Заодно — доводка интерфейса: чистая карточка
проекта, разделители в панели быстрого доступа и сгруппированное меню «Модули».

### ✨ Добавлено
- **Задачи — модуль правого слота.** Список TODO переехал из всплывающего окна карточки в отдельную панель
  справа (открывается из «Модули», палитры `Ctrl+K` и квикбара), как Git или «Контекст», и следует за активным
  проектом. Интерфейс переосмыслен: читаемые карточки (статус, важность, текст во всю ширину), фильтр
  **Активные / Все / Готово** со счётчиками, новая задача — через окно с полноценным полем-описанием.
- **Две вкладки: «Проект» и «Общие».** У каждого проекта свой список задач, плюс отдельный список **общих**
  задач вне проектов. Выбранная вкладка запоминается между запусками.
- **Перенос задач между списками** — кнопки «В общие» (из проекта) и «В проект» (из общих, в активный проект).
- **Экспорт и импорт задач в JSON.** Выгрузить можно текущий список или **все задачи сразу** (по проектам +
  общие, с разделением). При импорте — выбор режима: **заменить** (по умолчанию) или **добавить** к существующим.
- **Разделители в панели быстрого доступа.** В «Модули → Настройка панели» можно добавлять вертикальные
  разделители (сколько угодно, в любое место) и группировать кнопки на полосе под терминалом.

### ♻️ Изменено
- **Карточка проекта — чище.** Кнопки вивера и Git убраны с карточки (они есть в панели быстрого доступа под
  терминалом), скрывающаяся по наведению полоса действий тоже убрана — остались только ★ «избранное» и ⋮ «меню».
- **Меню «Модули» — группировка.** Пункты стали двухстрочными (название + описание помельче), а длинный список
  свёрнут в раскрывающиеся подменю **«Встроенные»** и **«Мои модули»** + отдельный пункт «Настройка панели».
- Панель задач по умолчанию шире — длинные формулировки читаются целиком.

- **Модуль «Контейнеры» — Docker и Podman прямо в редакторе** (замена десктопным GUI). Открывается справа,
  вкладки **Docker | Podman** с автоопределением и показом версий (`docker` / `docker compose` /
  `docker-compose` и аналогично для podman; если движок не установлен — так и написано). Разделы-аккордеоны:
  - **Контейнеры** сгруппированы по **compose-проектам** — сворачиваемые группы с цветным градиентом,
    **сортировкой** (стрелками, с запоминанием порядка) и **групповыми действиями** (старт / пауза / стоп /
    удалить всю группу разом).
  - **Поды** (для Podman), **Образы**, **Тома** и полоска **занятого места на диске** (`system df`).
  - Управление каждым объектом из UI: старт / пауза / возобновление / перезапуск / стоп / удаление.
  - **Клик по контейнеру** открывает детальный вид прямо в панели: **живые логи** (стрим `logs -f`) и
    **встроенный интерактивный терминал внутрь контейнера** (`exec`, сам выбирает bash или sh).
- **Git стал боковой панелью в стиле JetBrains** (вместо модального окна). Ветки и переключение, список
  изменений с **диффом по клику**, поле коммита (Commit / Commit & Push / Fetch / Pull / Push), откат файла
  и **история коммитов**. Привязан к выбранному проекту; при переключении проектов панель следует за ним.
- **Единый правый слот для модулей.** Вспомогательные панели — **Код** (вивер), **Git**, **Контейнеры** —
  открываются по одной за раз справа от терминала и собраны в меню **«Модули»**. Терминал остаётся главным.

### 🔧 Изменено
- **Меню приложения причёсано:** осталось **Файл · Настройки · Модули · О программе**. «Логи» переехали в
  «Файл»; «OpenRouter» и «Пульт» — в «Настройки».

## [1.0.100] — 2026-06-03

### 🐛 Исправлено
- **Окно не возвращало прежний размер после закрытия вивера** (#3). Если окно было растянуто почти на
  весь экран, открытие вивера упиралось ростом в край экрана (окно почти не расширялось), а закрытие
  всё равно ужимало его на полную ширину панелей — окно становилось **уже исходного**, и его приходилось
  растягивать заново. Теперь рост и сжатие симметричны: окно возвращается ровно к тому размеру, что был
  до открытия вивера. Ручное изменение размера окна между этими действиями корректно учитывается. То же
  касается системного («скретч») терминала. Спасибо @Eurgen за репорт.

## [1.0.98] — 2026-06-03

### ✨ Добавлено
- **Проверка обновлений.** Редактор сам сверяется с последним релизом на GitHub и, если вышла новая
  версия, показывает **плашку рядом с номером версии** в заголовке — клик открывает страницу загрузки.
  Авто-проверка происходит тихо вскоре после запуска (при отсутствии связи или актуальной версии ничего
  не появляется). Ручная проверка — кнопка **«Проверить обновление»** в окне «О программе». Это только
  уведомление: ничего не скачивается и не устанавливается само (формат дистрибутива `.deb`/portable-`.zip`
  не позволяет тихое обновление без прав администратора) — обновление ставится вручную, как обычно.

## [1.0.97] — 2026-06-03

### 🐛 Исправлено
- **Windows-сборка не запускалась** — на первом же старте падала с `Error: Cannot find module
  './lib/safe-name'`. Новые модули `lib/safe-name.js` (анти-traversal) и `lib/shell.js` (выбор оболочки),
  появившиеся в v1.0.95–96, не были добавлены в whitelist упаковки (`build.files` в `package.json`), поэтому
  в Windows-дистрибутив (asar) не попадали. Добавлено `lib/**` — теперь весь каталог `lib/` входит в сборку.
  На Linux проблема не воспроизводилась (там запуск из исходников). Спасибо за репорт первого Windows-теста.

## [1.0.96] — 2026-06-03

### ✨ Добавлено
- **Выбор оболочки терминала в настройках** (#2). На **Windows** по умолчанию теперь **PowerShell**
  (PowerShell 7 `pwsh.exe`, если установлен, иначе Windows PowerShell) с флагом `-NoLogo` — и при этом
  загружается пользовательский `$PROFILE` с алиасами (без `-NoProfile`); плюс выбор `cmd` или произвольного
  пути к оболочке. На **Linux** — `bash` по умолчанию или свой путь (zsh/fish/…). Раньше на Windows всегда
  запускался `cmd` (`resolveShell` безусловно брал `COMSPEC`, фолбэк на PowerShell был недостижим).
  Спасибо @Eurgen за детальный разбор.

## [1.0.95] — 2026-06-03

### ✨ Добавлено
- **Удалённый пульт (Android).** Управление редактором с планшета/телефона через self-hosted релей:
  полноценный терминал со своей экранной клавиатурой, переключение проектов и вкладок, просмотр и
  скачивание файлов с ПК, создание папок, безопасный перезапуск редактора. APK прилагается к релизу.
- **Одобрение устройства на ПК (pairing).** Новое устройство не получает доступ к терминалу, пока его не
  одобрят в редакторе на ПК (со сверкой кода) — даже при верном логине/пароле.
- **«Выйти на всех устройствах»** в меню «Пульт» — отзыв всех одобренных устройств и сессий на случай
  потери планшета.

### 🔒 Безопасность
- **Релей с аккаунтами:** вход по логину/паролю (PBKDF2), криптографическая изоляция аккаунтов,
  **защита от перебора** пароля (lockout), **отзывные сессии** с коротким TTL, лимиты на частоту и размер
  сообщений, троттлинг регистрации. Покрыто автотестами.
- **Анти-traversal** в файловых операциях (имя из недоверенного источника не может выйти за каталог).

## [1.0.53] — 2026-05-31

### 🐛 Исправлено
- **Эмодзи больше не «наезжают» на текст в терминале** (#1). xterm.js по умолчанию использует таблицы
  ширин Unicode V6, где нет новых emoji-диапазонов (📁 `U+1F4C1`, ⏰ `U+23F0` и др.) — они считались
  шириной 1 вместо 2, из-за чего статус-строка Claude Code и другой вывод смещались. Терминал переключён
  на таблицы **Unicode 11** (аддон `@xterm/addon-unicode11`). Спасибо @Eurgen за детальный разбор.

## [1.0.52] — 2026-05-30

### ✨ Добавлено
- **Чат с моделями через OpenRouter по своим ключам.** Пункт меню **OpenRouter** → модалка управления
  ключами; один ключ = плашка-чат в отдельной секции «OpenRouter». Клик по плашке открывает чат вместо
  терминала.
- **Выбор модели** из OpenRouter с поиском, **ценой за 1M токенов** (вход/выход) и размером контекстного окна.
- **Ответы стримом**, разметка **Markdown** и **подсветка кода** (highlight.js), копирование сообщений и
  отдельных код-блоков.
- **Несколько сессий на ключ** (выпадающий список: новая / переключить / переименовать / удалить) и
  настраиваемое окно контекста **1–100** последних сообщений — привязано к сессии.
- **Сгенерированные изображения** показываются прямо в чате: клик увеличивает (lightbox), есть кнопка «Скачать».
- На плашке ключа — **израсходовано / лимит** с кнопкой обновления.

### 🔒 Безопасность
- HTTP-запросы к OpenRouter идут из main-процесса — **ключ не попадает в код фронтенда**.
- Ссылки и картинки из ответов модели проходят allowlist схем; ссылки открываются только по http(s).
