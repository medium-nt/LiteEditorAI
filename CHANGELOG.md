# Changelog

Все заметные изменения LiteEditorAI. Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
нумерация — [SemVer](https://semver.org/lang/ru/). Проект в стадии **alpha**.

## [1.0.107] — 2026-06-04

### ✨ Добавлено
- **Модуль «Удалённые хосты» (RemoteHost) — серверы и хостинги прямо в редакторе.** Открывается из меню «Модули».
  - **Профили подключений по `SSH` / `SFTP` / `FTP`**, сгруппированные по категориям; создание / правка / тест,
    вход **в один клик**.
  - **SSH — полноценный терминал на сервере** (живой shell), несколько сессий как **вкладки** с переключением
    (`Ctrl+PageUp/PageDown`, `Ctrl+Shift+W`). Вход **по паролю** или **по ключу из системы без пароля**
    (SSH-агент / `~/.ssh`, как `ssh user@host`). **Keepalive** с настраиваемым интервалом — чтобы соединение
    не рвалось при простое.
  - **Просмотр файлов** (кнопка-папка у подключения): дерево удалённых файлов по **SFTP** (для SSH/SFTP) или
    **FTP/FTPS** с навигацией, размерами и чтением содержимого (пока только для просмотра).
- **Миникарта кода в вивере (как в VSCode):** уменьшенная копия файла справа от кода с рамкой текущей области
  и кликом-прыжком. Включается/выключается кнопкой в шапке вивера, состояние запоминается.
- **Пункт «Вивер» в меню «Модули»** — открыть просмотр файлов активного проекта (как кнопка-«глаз» на карточке);
  при открытой категории OpenRouter показывает заглушку «нужно выбрать проект».

### 🔧 Изменено
- **Карточки проектов и категории — действия раскрываются по наведению (hover-reveal):** у карточек кнопки
  спрятаны за свёрнутой стрелкой в строке заголовка (наведение выезжает кнопки — список спокойнее и компактнее),
  у категорий кнопки управления тоже выезжают при наведении на шапку. Иконки проекта собраны под заголовком.

### 🔒 Безопасность
- **Секреты RemoteHost шифруются системным хранилищем ключей** (safeStorage) и не покидают бэкенд; режим «без
  пароля» использует системный SSH-ключ/агент и **ничего не хранит**; **FTPS проверяет сертификат** (доверие
  самоподписанному — отдельным явным флагом).

## [1.0.104] — 2026-06-03

### ✨ Добавлено
- **Модуль «Git» — новые операции:** **stash** (спрятать все изменения, включая неотслеживаемые) и
  **возврат stash**; **«откатить все правки»** одной кнопкой (отслеживаемые файлы; новые не трогаются);
  **клик по коммиту в истории копирует его хеш**.

### 🔧 Изменено
- **Карточки проектов стали компактнее:** три кнопки (вивер · заметки · Git) переехали из нижнего футера
  в строку заголовка — сразу после звезды «избранное». Разделительная линия и футер убраны, карточка ниже.
- **Кнопки в модуле «Git» — аккуратные, не массивные.** Большие кнопки заменены на компактные pill-кнопки
  с иконками; действия разложены на две ровные строки: *Commit · Commit & Push* и *Fetch · Pull · Push │ Stash · Pop*.

## [1.0.103] — 2026-06-03

### ✨ Добавлено
- **`Ctrl+Enter` в терминале — перенос строки без выполнения команды.** Раньше `Enter` всегда сразу
  запускал команду, а продолжить ввод на новой строке можно было только через `Alt+Enter`. Теперь
  `Ctrl+Enter` переносит ввод на следующую строку (продолжение команды): на Linux/macOS — через `\`+
  перевод строки (вторичное приглашение `>` в bash/zsh), на Windows — через `LF` (ConPTY/PSReadLine, `>>`).
  Работает и в проектных, и в системных терминалах. Спасибо за идею в Issues.

## [1.0.102] — 2026-06-03

### ✨ Добавлено
- **Модуль «Базы данных» — лёгкий клиент Postgres / MySQL · MariaDB / SQLite** (замена DataGrip для рутины),
  встроенные драйверы (ставить ничего не нужно). Открывается из меню «Модули».
  - **Подключения по хосту или через SSH-туннель**, сгруппированы по категориям (сворачиваемые аккордеоны
    с градиентом, по умолчанию «Все»). Создание/правка/тест соединения, флаг **«только чтение»**.
  - **Дерево схемы** (схемы → таблицы/представления), **просмотр данных** таблицы в гриде с пагинацией.
  - **SQL-консоль** с подсветкой синтаксиса и выполнением по `Ctrl+Enter`; результат в гриде.
  - **Экспорт** результата/таблицы в **CSV / JSON / SQL** и **импорт** SQL-файла.
  - **Безопасность:** пароли (БД и SSH) шифруются системным хранилищем ключей и не покидают бэкенд; режим
    «только чтение» запрещает изменяющие запросы **на стороне сервера**; TLS по умолчанию с проверкой
    сертификата (доверие самоподписанному — отдельным явным флагом).
- **Системный терминал стал модулем с вкладками.** Несколько независимых системных шеллов (вне проектов,
  домашняя папка) — вкладки `＋` / закрытие / переименование, хоткеи `Ctrl+Shift+T` / `Ctrl+Shift+W`,
  `Ctrl+PageUp/PageDown`. Открывается из меню «Модули» или кнопкой в шапке.

### 🔧 Изменено
- **Категория OpenRouter** в колонке слева теперь видна **всегда** (даже без ключей — со счётчиком «0» и
  кнопкой настройки) и визуально отделена полоской «интеграции» от категорий проектов. Пункт «OpenRouter»
  убран из меню «Настройки» — управление ключами через шестерёнку прямо в секции.
- **Заметки — освежённый вид карточек:** заметная кнопка «В терминал», аккуратные иконки действий, метка
  очереди сверху. Функциональность не изменилась.

## [1.0.101] — 2026-06-03

### ✨ Добавлено
- **Модуль «Контейнеры» — Docker и Podman прямо в редакторе** (замена десктопным GUI). Открывается справа,
  вкладки **Docker | Podman** с автоопределением и показом версий (`docker` / `docker compose` /
  `docker-compose` и аналогично для podman; если движок не установлен — так и написано). Разделы-аккордеоны:
  - **Контейнеры** сгруппированы по **compose-проектам** — сворачиваемые группы с цветным градиентом,
    **сортировкой** (стрелками, с запоминанием порядка) и **групповыми действиями** (старт / пауза / стоп /
    удалить всю группу разом).
  - **Поды** (для Podman), **Образы**, **Тома** и полоска **занятого места на диске** (`system df`).
  - Управление каждым объектом из UI: старт / пауза / возобновление / перезапуск / стоп / удаление.
  - **Клик по контейнеру** открывает детальный вид прямо в панели: **живые логи** (стрим `logs -f`) и
    **встроенный интерактивный терминал внутрь контейнера** (`exec`, сам выбирает bash или sh).
- **Git стал боковой панелью в стиле JetBrains** (вместо модального окна). Ветки и переключение, список
  изменений с **диффом по клику**, поле коммита (Commit / Commit & Push / Fetch / Pull / Push), откат файла
  и **история коммитов**. Привязан к выбранному проекту; при переключении проектов панель следует за ним.
- **Единый правый слот для модулей.** Вспомогательные панели — **Код** (вивер), **Git**, **Контейнеры** —
  открываются по одной за раз справа от терминала и собраны в меню **«Модули»**. Терминал остаётся главным.

### 🔧 Изменено
- **Меню приложения причёсано:** осталось **Файл · Настройки · Модули · О программе**. «Логи» переехали в
  «Файл»; «OpenRouter» и «Пульт» — в «Настройки».

## [1.0.100] — 2026-06-03

### 🐛 Исправлено
- **Окно не возвращало прежний размер после закрытия вивера** (#3). Если окно было растянуто почти на
  весь экран, открытие вивера упиралось ростом в край экрана (окно почти не расширялось), а закрытие
  всё равно ужимало его на полную ширину панелей — окно становилось **уже исходного**, и его приходилось
  растягивать заново. Теперь рост и сжатие симметричны: окно возвращается ровно к тому размеру, что был
  до открытия вивера. Ручное изменение размера окна между этими действиями корректно учитывается. То же
  касается системного («скретч») терминала. Спасибо @Eurgen за репорт.

## [1.0.99] — 2026-06-03

### ✨ Добавлено
- **Иконка приложения у пульта (Android).** APK теперь устанавливается с собственной иконкой в фирменном
  стиле редактора (тёмный градиент + синий шеврон `>` и зелёный курсор `_`) вместо системной заглушки.
  Адаптивная иконка для Android 8+ (с монохромным слоем под тематические иконки Android 13+) и обычные

## [1.0.98] — 2026-06-03

### ✨ Добавлено
- **Проверка обновлений.** Редактор сам сверяется с последним релизом на GitHub и, если вышла новая
  версия, показывает **плашку рядом с номером версии** в заголовке — клик открывает страницу загрузки.
  Авто-проверка происходит тихо вскоре после запуска (при отсутствии связи или актуальной версии ничего
  не появляется). Ручная проверка — кнопка **«Проверить обновление»** в окне «О программе». Это только
  уведомление: ничего не скачивается и не устанавливается само (формат дистрибутива `.deb`/portable-`.zip`
  не позволяет тихое обновление без прав администратора) — обновление ставится вручную, как обычно.

## [1.0.97] — 2026-06-03

### 🐛 Исправлено
- **Windows-сборка не запускалась** — на первом же старте падала с `Error: Cannot find module
  './lib/safe-name'`. Новые модули `lib/safe-name.js` (анти-traversal) и `lib/shell.js` (выбор оболочки),
  появившиеся в v1.0.95–96, не были добавлены в whitelist упаковки (`build.files` в `package.json`), поэтому
  в Windows-дистрибутив (asar) не попадали. Добавлено `lib/**` — теперь весь каталог `lib/` входит в сборку.
  На Linux проблема не воспроизводилась (там запуск из исходников). Спасибо за репорт первого Windows-теста.

## [1.0.96] — 2026-06-03

### ✨ Добавлено
- **Выбор оболочки терминала в настройках** (#2). На **Windows** по умолчанию теперь **PowerShell**
  (PowerShell 7 `pwsh.exe`, если установлен, иначе Windows PowerShell) с флагом `-NoLogo` — и при этом
  загружается пользовательский `$PROFILE` с алиасами (без `-NoProfile`); плюс выбор `cmd` или произвольного
  пути к оболочке. На **Linux** — `bash` по умолчанию или свой путь (zsh/fish/…). Раньше на Windows всегда
  запускался `cmd` (`resolveShell` безусловно брал `COMSPEC`, фолбэк на PowerShell был недостижим).
  Спасибо @Eurgen за детальный разбор.

## [1.0.95] — 2026-06-03

### ✨ Добавлено
- **Удалённый пульт (Android).** Управление редактором с планшета/телефона через self-hosted релей:
  полноценный терминал со своей экранной клавиатурой, переключение проектов и вкладок, просмотр и
  скачивание файлов с ПК, создание папок, безопасный перезапуск редактора. APK прилагается к релизу.
- **Одобрение устройства на ПК (pairing).** Новое устройство не получает доступ к терминалу, пока его не
  одобрят в редакторе на ПК (со сверкой кода) — даже при верном логине/пароле.
- **«Выйти на всех устройствах»** в меню «Пульт» — отзыв всех одобренных устройств и сессий на случай
  потери планшета.

### 🔒 Безопасность
- **Релей с аккаунтами:** вход по логину/паролю (PBKDF2), криптографическая изоляция аккаунтов,
  **защита от перебора** пароля (lockout), **отзывные сессии** с коротким TTL, лимиты на частоту и размер
  сообщений, троттлинг регистрации. Покрыто автотестами.
- **Анти-traversal** в файловых операциях (имя из недоверенного источника не может выйти за каталог).

## [1.0.53] — 2026-05-31

### 🐛 Исправлено
- **Эмодзи больше не «наезжают» на текст в терминале** (#1). xterm.js по умолчанию использует таблицы
  ширин Unicode V6, где нет новых emoji-диапазонов (📁 `U+1F4C1`, ⏰ `U+23F0` и др.) — они считались
  шириной 1 вместо 2, из-за чего статус-строка Claude Code и другой вывод смещались. Терминал переключён
  на таблицы **Unicode 11** (аддон `@xterm/addon-unicode11`). Спасибо @Eurgen за детальный разбор.

## [1.0.52] — 2026-05-30

### ✨ Добавлено
- **Чат с моделями через OpenRouter по своим ключам.** Пункт меню **OpenRouter** → модалка управления
  ключами; один ключ = плашка-чат в отдельной секции «OpenRouter». Клик по плашке открывает чат вместо
  терминала.
- **Выбор модели** из OpenRouter с поиском, **ценой за 1M токенов** (вход/выход) и размером контекстного окна.
- **Ответы стримом**, разметка **Markdown** и **подсветка кода** (highlight.js), копирование сообщений и
  отдельных код-блоков.
- **Несколько сессий на ключ** (выпадающий список: новая / переключить / переименовать / удалить) и
  настраиваемое окно контекста **1–100** последних сообщений — привязано к сессии.
- **Сгенерированные изображения** показываются прямо в чате: клик увеличивает (lightbox), есть кнопка «Скачать».
- На плашке ключа — **израсходовано / лимит** с кнопкой обновления.

### 🔒 Безопасность
- HTTP-запросы к OpenRouter идут из main-процесса — **ключ не попадает в код фронтенда**.
- Ссылки и картинки из ответов модели проходят allowlist схем; ссылки открываются только по http(s).
